## 🖥️ 앱 접속 방법

1. 크롬북(또는 PC)에서 **크롬 브라우저**를 엽니다.
2. 주소창에 앱 주소를 입력합니다.
3. 화면에서 **우리 반(도시)을 선택** → 번호 선택 → PIN 입력하면 로그인됩니다.

> ⚠️ 선생님은 학생 로그인 화면 하단의 **"선생님 로그인"**을 눌러 학급비번으로 접속합니다.

---

## ✅ 처음 셋팅 (수업 시작 전 1회만)

### 1단계 — 선생님 계정으로 로그인

1. 앱 접속 → 화면 아래 **"선생님 로그인"** 클릭
2. 우리 반 **학급비번 7자리** 입력
3. 자동으로 선생님(시장) 화면으로 이동

---

### 2단계 — 학생 계정 만들기

1. 선생님 화면 → **"학생 관리"** 메뉴 클릭
2. **"학생 일괄 생성"** 버튼 클릭
3. 학생 수(예: 25명)를 입력하고 **생성** 버튼 클릭
4. 완료! 1번~25번 학생 계정이 자동으로 만들어집니다.

> 💡 계정을 만들면 학생은 자신의 번호와 PIN 4자리로 로그인합니다.  
> **PIN은 학생이 처음 로그인할 때 직접 설정하게 됩니다.**

---

### 3단계 — 학생들에게 안내하기

학생에게 알려줄 내용:
- 앱 주소
- 우리 반 도시 이름 (예: 수원시)
- 자기 번호 (출석번호)
- "처음 로그인하면 PIN 4자리를 직접 정하세요"

## 🌱 시흥시 테스트 반으로 먼저 연습하기

**수업 전에 선생님이 직접 체험해보고 싶을 때 사용합니다.**

1. 앱 접속 → **"시흥시"** 선택
2. "선생님용 테스트 반입니다. 선생님이신가요?" 화면에서 **"네, 맞아요"** 클릭
3. 번호 선택 → PIN 설정하면 학생 역할로 체험 가능

**선생님(시장) 역할로 체험하려면:**
- 화면 하단 **"선생님 로그인"** → 학급비번 **3643410** 입력

> ✅ 테스트 반 데이터는 실제 수업 반(수원·이천 등)과 완전히 분리됩니다.  
> 마음껏 눌러보고 실험해도 실제 수업에 영향을 주지 않습니다.

## 📅 수업 단계별 진행 방법

수업은 **총 5단계**로 진행됩니다. 교사가 단계를 직접 열고 닫습니다.  
**학생들은 선생님이 연 단계의 기능만 사용할 수 있습니다.**

---

### ⓪ 0단계 — 도시 탐구 (1차시)

**학생이 하는 것:**
- 경기도 지도에서 우리 도시 찾기
- 우리 도시의 특산품, 자원, 강점 조사하기
- 우리 도시 한 줄 소개 작성하기

**선생님이 할 것:**
1. 선생님 화면 → **"단계 설정"** → **"⓪ 도시탐구 열기"** 클릭
2. 학생들이 도시 탐구를 마치면 → **"수업 게시판"**에서 조사 내용 확인
3. 피드백을 남길 수 있습니다.

---

### ① 1단계 — 창업 (2~4차시)

**학생이 하는 것:**
- 사업계획서 작성 (회사명, 판매 상품, 판매 대상, 이유)
- 선정된 학생 → 대표이사(CEO) 역할 부여받음
- 탈락한 학생 → 취업 지원서 작성 → CEO가 채용
- 채용 안 된 학생 → 공무원으로 임명

**선생님이 할 것:**
1. **"단계 설정"** → **"① 창업 열기"**
2. **"사업계획서 검토"** 메뉴에서 학생들이 낸 계획서 검토
3. 마음에 드는 계획서 **"선정"** 클릭 → 해당 학생이 자동으로 CEO가 됨
4. CEO가 직원을 채용하고 남은 학생은 **"학생 관리"**에서 공무원으로 임명
5. **"급여 지급"** 버튼 클릭 → 역할별 일당 자동 지급

> 💰 **급여 기준:** CEO 11,000원 / 공무원 10,400원 / 직원 10,400원 (수업일 1회, 2026년 최저시급 기준)

---

### ② 2단계 — 생산 (4차시)

**학생이 하는 것:**
- CEO/직원: 업무일지 작성 (오늘 만든 것, 한 일 기록)
- CEO: 품의서(구매 신청서) 작성 → 선생님께 제출
- 공무원: 업무일지 + 기업 시찰 보고서 작성

**선생님이 할 것:**
1. **"단계 설정"** → **"② 생산 열기"**
2. **"품의서 결재"** 메뉴에서 학생이 신청한 구매 건 확인
   - 학생이 쓴 **물건 이름, 용도, 수량, 금액** 모두 표시됨
   - **✓ 승인** 또는 **반려** 클릭
3. **"급여 지급"** 클릭 (매 수업일)

> 📦 **품의서:** 학생이 필요한 재료를 신청하는 문서입니다.  
> 승인하면 회사 잔액에서 자동 차감됩니다.

---

### ③ 3단계 — 교류 (2차시)

**학생이 하는 것:**
- CEO: 교류카드 등록 (우리 회사가 제공할 것 / 원하는 것)
- 공무원: 다른 반 방문 → 교류 성사 기록
- 직원: 업무일지 작성

**선생님이 할 것:**
1. **"단계 설정"** → **"③ 교류 열기"**
2. **"종합 모니터링"**에서 교류 진행 상황 확인
3. **"급여 지급"** 클릭

---

### ④ 4단계 — 판매 (2차시)

**학생이 하는 것:**
- CEO: 회사 관리 → 상품 등록 → **판매대 QR 코드** 화면 보여주기
- 구매자(다른 학생): 앱 → "물건 사기" → QR 스캔 → 상품 선택 → PIN 입력 → 결제

**선생님이 할 것:**
1. **"단계 설정"** → **"④ 판매 열기"**
2. **"종합 모니터링"**에서 각 회사 잔액·매출 확인
3. **"급여 지급"** 클릭
4. 수업 마무리 후 결과 정산 확인

> 🛒 **QR 결제 방법 (학생 안내용):**  
> 1. 사고 싶은 학생: 앱 → 카드 → "물건 사기" 클릭  
> 2. 카메라로 판매 회사의 QR 스캔  
> 3. 상품과 수량 선택  
> 4. 내 PIN 4자리 입력 → 결제 완료

## 🖥️ 선생님 화면 주요 메뉴 안내

### 📊 종합 모니터링
- 학생 전체 현황을 한눈에 봅니다.
- **초록색**: 해당 단계 활동 완료 / **주황색**: 활동 중 / **빨간색**: 미활동
- 필터 버튼으로 **접속 중 / 다른 화면 / 미접속** 학생만 따로 볼 수 있습니다.

### 🧾 품의서 결재
- 학생이 제출한 구매 신청서를 검토합니다.
- 물건 이름, 용도, 수량, 금액이 모두 표시됩니다.
- **포기한 항목(기회비용)** 도 학생이 직접 쓴 이유와 함께 표시됩니다.
- ✓ 승인 또는 반려 클릭.

### 💸 급여 지급
- 매 수업일 1회 클릭하면 역할별 일당이 자동 지급됩니다.
- 같은 날 두 번 눌러도 중복 지급되지 않습니다.

### 📋 수업 게시판
- 학생별 활동 현황, 개념 이해도, 공무원 시찰 보고서를 확인합니다.
- 시찰 보고서에 **✓ 확인** 클릭으로 검토 완료 표시를 남길 수 있습니다.

### 👥 학생 관리
- 학생 계정 생성, 역할 변경(공무원 임명 등), PIN 초기화를 합니다.
- PIN을 잊은 학생이 있으면 여기서 초기화해 주세요.

## ❓ 자주 묻는 질문 (FAQ)

**Q. 학생이 PIN을 잊어버렸어요.**  
A. 선생님 화면 → 학생 관리 → 해당 학생 → "PIN 초기화" 클릭. 학생이 다시 새 PIN을 설정하게 됩니다.

**Q. 단계를 잘못 열었어요. 되돌릴 수 있나요?**  
A. 단계 설정에서 이전 단계로 되돌릴 수 있습니다. 단, 학생이 이미 입력한 데이터는 유지됩니다.

**Q. 코드를 잘못 입력했는데 데이터가 사라졌나요?**  
A. 아닙니다. 코드(앱)와 데이터(DB)가 완전히 분리되어 있어서 앱이 업데이트되어도 데이터는 절대 사라지지 않습니다.

**Q. 여러 반이 동시에 수업해도 되나요?**  
A. 네. 각 반 데이터는 완전히 분리되어 있어서 서로 영향을 주지 않습니다.

**Q. 학생이 다른 반 학생으로 로그인할 수 있나요?**  
A. 불가능합니다. 반 선택 후 해당 반 번호와 PIN이 정확히 일치해야만 로그인됩니다.

**Q. 판매 단계에서 QR 결제가 안 돼요.**  
A. 상품이 등록되어 있는지 확인하세요. CEO가 회사 관리 → 상품 추가를 먼저 해야 합니다.

**Q. 공무원은 몇 명으로 지정하면 되나요?**  
A. 학급 규모에 따라 자유롭게 지정하면 됩니다. 보통 2~3명을 추천합니다.

## 📌 수업 흐름 한눈에 보기

```
[수업 전] 선생님 로그인 → 학생 계정 생성 → 학생에게 안내
    ↓
[0단계 탐구] 단계 열기 → 학생 도시 조사 → 게시판 확인
    ↓
[1단계 창업] 단계 열기 → 사업계획서 검토·선정 → 직원 채용 → 공무원 임명 → 급여 지급
    ↓
[2단계 생산] 단계 열기 → 업무일지 확인 → 품의서 결재 → 급여 지급
    ↓
[3단계 교류] 단계 열기 → 교류 활동 → 모니터링 → 급여 지급
    ↓
[4단계 판매] 단계 열기 → QR 결제 활동 → 매출 확인 → 급여 지급 → 정산
```
