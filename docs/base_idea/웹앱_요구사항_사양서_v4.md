## 0. 한 줄 요약

초등 4학년 사회 「경제활동과 지역 간 교류」 단원을, **5개 반이 각자 운영하는 가상 창업 시뮬레이션 웹앱**으로 구현한다. 교사(시장)가 수업 단계를 통제하고, 학생은 역할(CEO·직원·공무원)별 기능으로 창업→생산→교류→판매를 경험하며, 모든 과정이 과정중심평가 데이터로 기록된다.

> **설계 철학**: 활동이 '재미'에 그치지 않고 '개념 형성'으로 이어지는 구조. 기능을 구현할 때 해당 개념 정착 장치를 반드시 함께 심는다.

| 역할 | 서비스 | 티어 | 비고 |
|---|---|---|---|

| QR 생성 | `qrcode` npm 패키지 | 무료 | — |
| QR 스캔 | `html5-qrcode` npm 패키지 | 무료 | 브라우저 카메라 |

- 유료 서비스·결제 모듈 일절 사용하지 않는다. (앱 내 화폐는 가상 모형돈)
- 무료 티어 한도는 빌드 전 현재 기준으로 확인할 것 (5개 반 × 25명 + 교사 5명 = 130명 규모)

### Supabase 7일 정지 우회 (keep-alive)

Supabase 무료 티어는 7일간 API 호출이 없으면 프로젝트를 자동 정지한다.
GitHub Actions cron으로 3일마다 API를 호출해 타이머를 리셋한다.

```yaml
# .github/workflows/keep-alive.yml
name: Keep Supabase Alive
on:
  schedule:
    - cron: '0 9 */3 * *'
jobs:
  ping:
    runs-on: ubuntu-latest
    steps:
      - run: curl ${{ secrets.SUPABASE_URL }}/rest/v1/ -H "apikey: ${{ secrets.SUPABASE_ANON_KEY }}"
```

GitHub 저장소 Settings → Secrets에 `SUPABASE_URL`, `SUPABASE_ANON_KEY` 추가 필요.

## 1-A. 이식성 설계 (다른 교사가 포크해서 독립 운영)

### 개요

이 앱은 **반당 1개 Supabase 프로젝트**를 독립적으로 운영한다.
다른 교사가 포크하면 자신만의 DB·Auth·데이터를 가진 독립 인스턴스를 30분 안에 구축할 수 있다.

### 저장소에 반드시 포함할 파일

```
프로젝트 루트/
 ├─ .env.example              ← 환경변수 템플릿 (값 없이 키만)
 ├─ SETUP.md                  ← 포크 후 세팅 가이드 (비개발자용)
 ├─ supabase/
 │   └─ migrations/           ← SQL 마이그레이션 파일 (테이블 생성·RLS 정책)
 │       ├─ 001_init.sql
 │       ├─ 002_rls.sql
 │       └─ ...
 └─ .github/
     └─ workflows/
         └─ keep-alive.yml    ← Supabase 정지 방지 cron
```

### 필요한 환경변수 (3개)

```env
# .env.example
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=
```

Vercel 대시보드 → Project Settings → Environment Variables에 입력.
GitHub Actions Secrets에도 `SUPABASE_URL`, `SUPABASE_ANON_KEY` 동일하게 추가.

### 포크 후 세팅 순서 (SETUP.md 내용 기준)

```
① GitHub에서 저장소 포크
② Supabase 계정 생성 → 새 프로젝트 생성 → URL·키 복사
③ Supabase SQL 에디터에서 migrations/ 파일 순서대로 실행
    (또는 Supabase CLI: npx supabase db push)
④ Vercel에서 "Import Git Repository" → 포크한 저장소 선택
⑤ Vercel 환경변수 3개 입력 → 배포
⑥ GitHub Secrets 2개 추가 (keep-alive용)
⑦ 첫 접속 → 교사 계정 생성 → 반·학생 명단 업로드
```

예상 소요 시간: **약 30분** (Supabase·Vercel·GitHub 계정 보유 기준)

## 2. 사용자 · 역할 (RBAC)

### 역할 5종

| 역할 | 소속 | 권한 요약 |
|---|---|---|
| **시장 (교사)** | 반 전체 | 단계 통제, 사업체 선정·지원금, 공무원 임명, 전 계좌 조회, 평가 데이터·CSV 내보내기 |
| **지원자** | 반 공통 | 사업계획서 작성·제출. 선정/채용 후 다른 역할로 전직. |
| **CEO (창업가)** | 기업 소속 | 회사 설립(이름·상품 등록), 직원 직접 채용, 예산 조회, 품의서 작성·제출, 직원 급여 지급, 회사 수금 QR 표시 |
| **직원** | 기업 소속 | 업무 기록(실물 제작 내용), 본인 급여·잔액 조회, QR로 다른 회사 제품 구매(자기 회사 구매 불가), 성찰·개념 작성 |
| **공무원** | 반 공통 (시청) | 공용 시설 목록·이용료 설정, 기업 시찰 보고서 작성, 예산/정체 경보 전달, 전체 거래 장부 열람, 교류 중개 기록, 급여 장부 집계, 성찰·개념 작성 |

### 한 반의 팀 구조 (약 25명 기준)

```
반(예: 이천시) — 25명
 ├─ 기업 A: CEO 1 + 직원 N   ┐
 ├─ 기업 B: CEO 1 + 직원 N   │ 4개 팀 (CEO 4 + 직원 약 16)
 ├─ 기업 C: CEO 1 + 직원 N   │
 ├─ 기업 D: CEO 1 + 직원 N   ┘
 └─ 시청: 공무원 2~3명 (company_id 없음, class 단위 소속)
```

- 반당 최대 4개 기업, 기업당 CEO 1명 + 직원 최대 4명 (총 5명/팀)
- **CEO가 직원을 직접 채용** (지원자 목록에서 선택)
- **공무원은 교사(시장)가 임명**
- 모든 학생은 최초 `applicant` → 선정/채용/임명 후 다른 역할로 전직

### 로그인 (4학년도 30초 안에 가능해야 함)

- 방식: **반 선택 → 번호 선택 → 4자리 PIN**
- 아이디/이메일 입력 없음. 타이핑 최소화.
- 교사는 별도 로그인 (이메일/비밀번호 또는 매직링크).
- 로그인 상태 유지(세션), 새로고침해도 유지.

### PIN 보안 정책

- 최초 로그인 시 기본 PIN `1234` → **강제 변경** (변경 전까지 다른 기능 사용 불가)
- `1234`로의 재변경 불허
- PIN 3회 연속 실패 시 **30초 쿨다운** (교실 내 선점 공격 방어)
- 교사가 학생 PIN을 `1234`로 초기화할 수 있음 (분실 대비)

### 학생 명단 · 계정

- 교사가 CSV 업로드 또는 화면 입력으로 계정 일괄 생성
- 업로드 항목: 번호, (선택)닉네임, 반, 역할, 소속 회사
- **표시 이름은 번호/닉네임** — 실명·생년월일 등 개인정보 미수집 (개인정보 최소화)

## 3. 수업 단계 제어 (핵심 안전장치)

- 반마다 **현재 단계(stage)** 상태가 하나 있다. **시장(교사)만 단계를 변경**할 수 있다.
- 단계: **5단계 — ⓿ 도시 탐구 ① 창업 ② 생산 ③ 교류 ④ 판매**
- 학생 화면은 **현재 단계에 열린 기능만 활성화**, 나머지는 잠금(🔒 + "N단계에 열려요" 안내).
- 단계 변경 시 해당 반 모든 학생 화면이 **실시간으로 반영** (Supabase Realtime).
- 교사가 다음 단계로 넘기기 전 진행 현황(예: "12명 중 9명 품의서 제출")을 확인하고 확인창을 거친다.
- **성찰·개념 작성 기능은 모든 단계에서 항상 열림** (평가 데이터 상시 수집).

## 4. 운영 범위 · 박람회 모드

- 기본은 **반 내부 운영** (5개 반이 서로 격리).

- 교류 기록: 양측이 주고받은 것 입력 → 공무원이 기록 → 장부에 저장 (교사 승인 큐).
- 박람회 모드가 꺼지면 다시 반 내부로 격리.

## 5. 핵심 기능 명세

### 5-0. 도시 탐구 (전원, 0단계)

- **지도에서 우리 도시 클릭 선택**
  - 경기도 시군 경계 SVG 지도 (통계청 공개 데이터 기반)
  - 오답 클릭 시 1.5초 쿨다운 (소거법 방지)
  - 지도 정답 확인 후에만 조사 내용 제출 가능
- **도시 자료 조사 입력**: 특산품·자원 / 우리 도시 강점 / 한 줄 소개 (앱이 답을 알려주지 않음)
- **반 워드클라우드 3종** (교사 송출용): 세 항목 각각 빈도에 따라 크기 변화
- **교사 단어 묶기**: 유사어 병합 (도자기/도자/이천도자기 → 도자기)
- **대표 자료 확정**: 교사가 워드클라우드에서 클릭 → 반 대표 특산품·강점·소개 확정 → 이후 사업계획서·교류·홍보에 자동 참조

### 5-1. GRASPS 인트로 스토리

**공통 인트로 3장 + 역할별 스토리 2장 슬라이드**

#### 공통 인트로 (최초 로그인 시 1회 자동 재생)

| 장면 | 내용 |
|---|---|
| 1 | "경기도 5개 도시가 경제 공동체를 만들기로 했습니다." → 상호의존 예고 |
| 2 | "창업 지원 프로그램이 시작됩니다." → 우리 도시 소개, 희소성·합리적 선택 예고 |
| 3 | "이제 여러분의 역할이 주어집니다." → 역할 예고, "지금부터 시작!" |

#### 역할별 스토리 (공통 인트로 직후, 역할 확인 후 재생)

- **지원자**: 창업 공고 발견 → 좋은 사업계획서 조건 안내
- **CEO (선정 후)**: 경영자의 하루 → CEO 할 일 목록 + 지원금 활용 안내
- **직원 (채용 후)**: 직원의 하루 → 업무일지·급여·소비 안내
- **공무원 (임용 후)**: 공무원의 역할 → 시설 관리·시찰·장부·보고 안내

#### 역할 전직 팝업 — 트리거 시점 역할별 차이

전체화면 축하 카드. **역할에 따라 팝업이 뜨는 시점이 다르다.**

| 역할 | 팝업 트리거 시점 | 방식 |
|---|---|---|
| **CEO** | 교사가 선정한 이후, **해당 학생의 다음 번 로그인 시** | `users.reveal_pending = 'ceo'` 플래그 → 로그인 시 소비 |
| **직원** | CEO가 채용 버튼을 클릭하는 **그 순간** | Supabase Realtime — `users.role` 변경 구독 → 즉시 팝업 |
| **공무원** | 교사가 임명 버튼을 클릭하는 **그 순간** | 동일하게 Realtime 구독 |

팝업 내용:
- **CEO**: "🎊 귀하의 사업계획서가 선정되었습니다. 지원금 100,000원이 입금됐어요." → CEO 인트로 재생
- **직원**: "📨 [회사명] 직원으로 채용되었습니다. 일급 3,000원." → 직원 인트로 재생
- **공무원**: "📜 [도시명] 시청 공무원으로 임용되었습니다. 일급 3,500원." → 공무원 인트로 재생

### 5-1-A. 사업계획서 + 역할 전직 흐름

1. **전원 작성**: 모든 학생이 1단계 진입 시 사업계획서 작성·제출 가능
2. **문서형 양식** (한 화면 문서 형식):
   - 회사 이름
   - 무엇을 만들/팔까요?
   - 누구에게 팔까요? (타겟)
   - 우리 도시 자원·특징 활용 여부 체크박스 (선정 가점) + 구체적 설명
   - 선정되어야 하는 이유 (설득 칸)
   - 구입 예정 품목 표 (품목명 / 수량 / 단가 / 소계 자동계산)
   - **예비비 설정**: "총 지원금 중 예비비로 남길 금액" 슬라이더·입력 → 나머지를 재료비로 사용할 계획 금액에 자동 반영
3. **시장 심사**: 제출된 계획서가 교사 화면에 카드로 나열. 최대 4팀 선정.
4. **역할 자동 전직**: 선정 → `role = ceo`, `reveal_pending = 'ceo'` → 해당 학생 **다음 로그인 시** CEO 전직 팝업
5. **선정 취소 가능**: 교사가 실수로 선정했을 때 "선정 취소" → 지원자로 복귀, 지원금 회수 (멱등 처리)
6. **사업계획서 수정 기회 1회**: 2단계(생산) 시작 후 CEO에게 수정 기회 1회 제공. 수정 이력 기록.
7. **지원금 자동 지급**: 선정 → 정부 계좌 → 회사 계좌 이체. 특산품 연계 체크 시 교사 승인 후 보너스 추가. **회사당 1회만** (멱등).
8. **미선정 계획서 보존**: 평가 포트폴리오로 보존.

### 5-1-B. 직원 채용 흐름

#### 채용 주체 분리
- **CEO가 직원을 직접 채용**: CEO 홈 화면 "직원 채용" 메뉴 → 지원자 목록에서 선택 → 채용 클릭
  - 채용 인원: CEO 포함 최대 5명/팀 (직원 최대 4명)
  - 채용 정원 초과 시 버튼 자동 비활성화
  - 채용 즉시 Realtime으로 직원 전직 팝업 표시
- **공무원은 교사(시장)가 임명**: 시장 통제실 "미배정 지원자" 목록에서 공무원 임명 가능
  - 임명 즉시 Realtime으로 공무원 전직 팝업 표시

#### 미채용 방치 방지
- 채용 마감 시각(교사 설정) 도래 시 미배정 학생을 교사 화면에 경고 표시
- 교사가 직권으로 미배정 학생을 직원·공무원으로 긴급 배정 가능
- 모든 학생이 CEO·직원·공무원 중 하나로 배정 완료되면 `applicant` 역할은 소멸

### 5-2. 예산 · 지갑 (전 역할)

#### 임금 체계 (확정)

| 역할 | 일급 | 8회 수업일 누적 최대 |
|---|---|---|
| CEO | 4,500원 | 36,000원 |
| 공무원 | 3,500원 | 28,000원 |
| 직원 | 3,000원 | 24,000원 |

- 수업일(교사가 "수업 열기" 버튼을 눌러야만 당일 급여 활성화)
- 수업일당 1회 지급 (중복 지급 불가 — 멱등 처리)
- CEO 급여: 본인 업무일지 작성 후 회사 계좌에서 자동 차감
- 직원 급여: 업무일지 제출 → CEO 확인 → 지급 (미확인 N일 경과 시 교사 화면 경고)
- 공무원 급여: 업무일지(시찰 보고서 포함) 제출 → 시장 확인 → 지급

#### 회사 예산 구조

| 항목 | 금액 | 비고 |
|---|---|---|
| 기본 지원금 | 100,000원 | 선정 시 자동 지급 |
| 특산품 연계 보너스 | +20,000원 | 교사 승인 시 추가 |
| 재료비 (권장) | ~60,000원 | 품목 표 합계 |
| 시설이용료 (권장) | ~20,000원 | 공용 시설 이용 |
| 예비비 | 학생이 직접 설정 | 사업계획서 단계에서 결정 |

- 판매 상품 단가 권장: 2,000~5,000원
- 학생 최대 누적 임금 24,000원 → 5~12개 구입 가능 (충분한 구매 경험)

### 5-3. 회사 설립 · 상품 관리 (CEO, 1단계~)

선정 직후부터 언제든지 수정 가능. 영업 시작 전 필수 완료.

- **회사 아이콘** 선택 (이모지 팔레트)
- **회사 이름 확정** (최대 20자, 욕설·실명 필터 + 교사 수정 가능)
- **판매 상품 목록** 등록:
  - 상품명 (최대 15자) / 초기 재고 수량 / 판매 단가
  - 상품 추가·삭제 (최대 8종)
  - 단가는 양의 정수만 허용 (서버 검증)
- **항상 수정 가능**: 매진 표시(재고 0), 가격 변경(세일), 신상품 추가 허용
  - 가격 변경 시 장부에 이력 기록
- 미등록 상태에서 QR 결제 시도 시 차단

### 5-4. 공용 시설 관리 (공무원, 1단계~)

공무원이 시설 목록·이용료를 직접 설정해야 기업이 이용 가능.

- **시설 목록 추가·편집** (최대 8종): 시설명 / 이용 단위(회·시간·칸 등) / 단가
- 기업이 시설 이용 시 **계산서 화면**: 시설명 / 이용 내용 메모 / 수량 입력 → 수량 × 단가 = 총금액 자동계산 → CEO PIN 입력 → 회사 계좌 → 시청 계좌 이체
- 시청 계좌 잔액은 공무원 홈에서 항상 표시
- 공무원이 시설 미등록 시 기업 이용 메뉴 비활성화 + 안내

### 5-5. 품의서 / 물품 구입 (CEO, 1단계)

- 품목·수량·단가·구입처·필요 이유 입력, 합계·잔액 자동 계산
- **"사고 싶었지만 뺀 항목 + 이유" 칸** (= 기회비용, 평가 자료)
- 사업계획서에서 설정한 예비비는 "남겨야 할 최소 잔액"으로 화면에 표시
- 제출 → 시장 결재 → 승인 시 회사 계좌에서 차감

### 5-6. 직원 업무 기록 (직원, 2단계~)

직원이 제작 활동 중 하는 실물 작업을 기록한다. 기록이 급여의 근거가 된다.

- **오늘 한 일**: 자유 서술 (예: "책갈피 코팅 5개 완성", "홍보 포스터 디자인 완료")
- **성실히 일한 여부 체크**: CEO가 확인 → 급여 지급
- 기록은 활동 타임라인으로 누적 → 과정중심평가 데이터

### 5-7. 공무원 역할 (확장 설계)

공무원은 반당 2~3명이며, 특정 기업 소속이 아닌 **반(class) 단위 행정관**이다.
앱이 자동화하지 못하는 **현장 관찰·보고** 역할을 담당해, 손으로 만드는 일이 서툰 학생도 의미있게 참여한다.

#### 앱 알림 → 공무원이 직접 움직이는 구조

```
앱 알림 발생 → 공무원 홈에 표시 → 해당 기업 찾아가 확인 → 시찰 보고서 작성 → 시장(교사)에게 제출
```

| 알림 종류 | 트리거 조건 (교사가 비율/시간 설정 가능) | 공무원 행동 |
|---|---|---|
| **예산 경보** | 기업 잔액이 초기 지원금의 설정 비율 이하로 하락 | 해당 기업 찾아가 경보 전달·상황 파악 |
| **생산 정체 경보** | N분 이상 업무 기록 없는 기업 | 시찰 후 보고서 작성 |
| **박람회 교류 신청** | 다른 반 기업이 교류 신청 | 해당 회사에 직접 전달·중개 |

#### 기업 시찰 보고서 (공무원 핵심 화면, 2단계~)

시찰할 때마다 작성하고 시장에게 제출한다.

- **시찰 기업·시각**: 자동 기록
- **현재 예산**: 앱이 자동으로 표시 (초기 대비 %)
- **생산 진행 상태**: 선택 (잘 되고 있음 / 느림 / 문제 있음)
- **내가 본 것**: 자유 서술 2~3줄 (현장 분위기·협업 상황·특이사항)
- **예산 경보 전달 완료 여부**: 체크박스
- **시장에게 한마디**: 한 줄 자유 입력

보고서는 교사 화면에 누적되어 앱 숫자로 잡히지 않는 **정성적 현황 데이터**로 활용된다. 교사의 실시간 현황 보드를 보완한다.

#### 거래 장부 열람 및 집계

- 반 전체 거래를 시간순으로 열람·필터 (회사별·종류별: 지원금/구매/급여/시설료/교류)
- 급여 장부 집계 열람

### 5-8. QR 결제 / 수금 (4단계, 체크카드 방식)

**확정된 결제 흐름**: 손님이 자신의 QR 카드(A4 인쇄·코팅한 체크카드)를 판매 부스 태블릿에 제시 → 판매자가 스캔 → 금액 확인 → 손님이 PIN 입력 → 완료.

- **QR 카드 = 체크카드**: 학생 1인 1장, A4 인쇄 후 코팅. 부스 태블릿 1대(판매자 단말) 배치.
- 카드 분실 시 교사가 수동 재발급 (보정 화면)
- 상품 목록 표시 → 손님이 선택 → "왜 샀나요?" 입력 → PIN 확인 → 결제
- **자가구매 차단**: 자기 회사 상품은 목록에서 제외 (서버 검증)
- **재고 초과 차단**: 재고 0 상품 구매 불가, 구매 즉시 재고 차감
- **잔액 부족 차단**: 잔액 부족 시 버튼 비활성화
- **이중처리 방지**: 버튼 디바운스 + 서버 멱등키 (거래 UUID 중복 거부)
- 서버에서 구매자 잔액 검증 → 양 계좌 동시 갱신 → 양쪽 화면 실시간 반영
- 영수증 자동 기록. "왜 샀는지" 입력 = 합리적 소비 평가 자료.

### 5-9. 급여 (2단계~)

- CEO만 자기 회사 직원에게 지급. 교사가 1일 급여 상한 설정 가능.
- 미승인 장기 방치 시 교사 화면에 경고 → 교사 직권 지급 가능
- 직원: 본인 급여·누적·잔액 조회. 공무원: 급여 장부 집계 열람.

### 5-10. 박람회 교류 (3단계, 교사가 켜야 활성)

- **판매 아님, 교류임**: "판매", "구매", "결제" 단어 제거. "제공", "요청", "협력 기록"만 사용.
- 다른 도시 홍보자료 열람 (확정 대표자료 자동 채움 + 제공 가능 항목·필요한 것)
- 교류 기록: 양측이 주고받은 것 입력 → 공무원이 기록 → 장부에 저장
- 금전 거래 없음 (숫자/금액 UI 미표시)
- 도움받은 기업 → 감사 카드 전달 → 가장 많이 받은 기업에 우수기업 인증 (자동 집계)

### 5-11. 성찰 · 개념 학습 (전원, 상시)

- 한 줄 답 + 기분 이모지 (가볍고 자주)
- 4단계 마무리 시 "최종 성찰 보고서(워크시트)" 접근 가능

## 5-12. 개념 정착 설계 (5가지 장치)

> **원칙**: 기능을 구현할 때 해당 개념 정착 장치를 함께 심는다. 질문을 너무 자주 띄우면 흐름이 끊긴다 — 아래 트리거를 엄수할 것.

### (0) 차시별 형성평가 — 활동 완료 이벤트 자동 트리거

| 앱 완료 이벤트 | 형성평가 개념 | 예시 문항 |
|---|---|---|
| 도시 탐구 제출 (0단계) | 특산품·지역 특성 | "특정 지역이 특히 잘 만드는 것은?" |
| 사업계획서 제출 (1단계) | 희소성·기회비용 | "하고 싶은 건 많은데 돈이 적을 때 이 상태는?" |
| 품의서 제출 (1단계) | 합리적 선택 | "사고 싶었지만 뺀 이유를 적어봐요" |
| 업무일지 최초 제출 (2단계) | 생산·소비 | "재료 사서 물건 만드는 것은 생산? 소비?" |
| 교류 기록 최초 저장 (3단계) | 교류·상호의존 | "없는 것을 다른 지역과 주고받는 것은?" |
| QR 결제 최초 완료 (4단계) | 합리적 소비 | "값·필요·품질을 따져 현명하게 사는 것은?" |

- 문항 형식: 객관식(2~4지선다) + 단답형 혼용. 틀리면 즉시 짧은 설명 → 재시도 1회.
- 동일 이벤트에 대한 형성평가는 1회만 (중복 트리거 방지).
- 교사가 문항을 수정·추가할 수 있는 문항 관리 화면 제공 (DB에 저장, 하드코딩 금지).

### (1) 활동 직후 개념 연결 한 컷 (micro-concept)

핵심 행동 완료 즉시 짧은 개념 질문 1개 자동 팝업.
- 품의서 제출 → "사고 싶던 걸 다 못 샀죠? 왜?" → **희소성**
- 재료 구입 → "생산일까 소비일까?"
- 교류 체결 → "왜 교류했나요? = **상호의존**"
- QR 결제 후 → "이 소비, 합리적이었나요?"

### (2) 단계 시작 = 핵심 질문, 단계 끝 = 내 말로 답하기

단계 진입 시 핵심 질문 표시 → 단계 종료 시 같은 질문에 내 말로 답 작성.

### (3) 단계별 쪽지시험 3회

| 회차 | 시점 | 확인 개념 |
|---|---|---|
| **1차** | 창업→생산 전환 직전 (7차시 후) | 경제활동·희소성·합리적 선택·기회비용 (5문항) |
| **2차** | 교류→판매 전환 직전 (11차시 후) | 생산·소비·지역 간 교류·상호의존 (5문항) |
| **최종** | 판매 단계 종료 후 (15차시) | 전체 통합 (5문항) + 핵심 문장 완성 |

- 교사가 "다음 단계로 넘기기" 클릭 시 "쪽지시험 먼저 치를까요?" 확인창
- 교사 화면에 반 전체 정답률 실시간 집계
- 점수가 아닌 "몇 개 이해했어요" 표시

### (4) 개념 관문 (단계당 1회)

단계 전환 직전 핵심 개념 1문제 확인. 틀려도 막지 않고 설명 후 재시도. 교사 현황 보드에 통과 현황 표시.

### (5) 내 행동이 곧 개념 사례 / 핵심 문장 완성

- 학생의 실제 행동 데이터를 개념 예시로 되돌려줌 ("코팅기를 포기하고 색종이를 샀지? 그 포기한 것이 **기회비용**")
- 단원 끝에서 빈칸 핵심 문장 완성: "자원은 (한정)되어 있어 (합리적)하게 선택하고, 지역은 (상호의존)하며 함께 발전한다."

## 11. 빌드 우선순위 (단계적 — MVP부터)

1. **MVP**: 로그인(반+번호+PIN, 최초 PIN 강제 변경) · 역할별 홈 · 단계 제어(실시간) · 계좌/거래 기본 · 교사 1순위 기능(실시간 현황 보드·거래취소·잔액보정·PIN 재발급·지금 할 일 카드)
2. **GRASPS 인트로**: 공통 인트로 3장 + 역할별 인트로 2장 + 역할 전직 팝업 3종
3. **도시 탐구**: 경기도 SVG 지도(쿨다운 포함) · 조사 입력 · 반 워드클라우드 3종 · 교사 단어 묶기 · 대표 자료 확정
4. **창업**: 사업계획서(문서형 + 예비비 + 특산품 체크 + 품목 표) · 교사 선정/취소 · 전직 팝업 · 지원금 멱등
5. **회사 설립 · 채용**: CEO 회사 설립(이름·상품 등록) · CEO 직원 채용(최대 4명) · 교사 공무원 임명 · 채용 후 사업계획서 수정 1회
6. **생산·급여**: 직원 업무일지(실물 제작 기록) · 급여(역할별 단가 분리) · 공무원 시설 목록 관리 · 시설 이용 계산서 · **공무원 시찰 보고서 · 예산/정체 경보 알림 · 교사 보고서 열람**
7. **판매·QR**: 상품 목록 기반 QR 결제(체크카드 방식, 자가구매·재고·잔액·이중처리 방지) · 돈 흐름 타임라인
8. **교류·박람회**: 재료·기술 교류 전용 UI("판매" 단어 제거) · 도시 홍보자료 · 교류 기록 · 감사 카드 · 우수기업 집계
9. **교사 2순위**: 전체 일시정지 · 공지 · 개입 알림 · 순회 평가 메모 · 다회차 되돌리기 · 미채용 경고 · 미승인 급여 경고
10. **평가·개념 정착**: 차시별 형성평가(6종 이벤트 연동) + micro-concept + 쪽지시험 3회 + 개념 관문 + 핵심 문장 완성 · 교사 대시보드 3탭(공무원 보고서 포함) · CSV 내보내기

각 단계가 끝날 때마다 동작 확인 후 다음으로 진행.
