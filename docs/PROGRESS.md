## 미완료 — 빌드 우선순위 순서

### 1순위: MVP (다음 작업)

> 목표: 로그인하고 역할별 홈 화면이 보이며, 교사가 단계를 바꾸면 학생 화면이 실시간으로 반응하는 상태

- [ ] Supabase 클라이언트 설정 (`lib/supabase/`)
- [ ] DB 마이그레이션: `001_init.sql` (핵심 테이블: classes, users, accounts)
- [ ] DB 마이그레이션: `002_rls.sql` (Row Level Security 정책)
- [ ] 로그인 화면: 반 선택 → 번호 선택 → PIN 4자리
- [ ] 최초 로그인 시 PIN 강제 변경 (1234 재설정 불허)
- [ ] PIN 3회 실패 → 30초 쿨다운
- [ ] 역할별 홈 화면 (시장·CEO·직원·공무원·지원자 각각 다름)
- [ ] 단계 제어: 교사가 단계 변경 → Supabase Realtime → 전체 학생 화면 즉시 반영
- [ ] 계좌 잔액 조회 (역할별)
- [ ] 교사 현황 보드: 반 진행 현황·잔액 요약
- [ ] 교사 기능: 거래 취소·잔액 보정·PIN 초기화·단계 되돌리기

### 2순위: GRASPS 인트로

- [ ] 공통 인트로 슬라이드 3장
- [ ] 역할별 인트로 슬라이드 2장
- [ ] 역할 전직 팝업 3종 (CEO: 다음 로그인 / 직원·공무원: Realtime 즉시)

### 3순위: 도시 탐구 (0단계)

- [ ] 경기도 SVG 지도 (시군 경계, 오답 1.5초 쿨다운)
- [ ] 도시 조사 입력 (특산품·강점·소개)
- [ ] 반 워드클라우드 3종 (교사 송출용)
- [ ] 교사 단어 묶기 (유사어 병합)
- [ ] 대표 자료 확정

### 4순위: 창업 (1단계)

- [ ] DB 마이그레이션: business_plans, companies
- [ ] 사업계획서 (문서형: 예비비 슬라이더·특산품 체크·품목 표)
- [ ] 교사 선정/취소 + 지원금 자동 지급 (멱등)
- [ ] 역할 전직 팝업 연동

### 5순위: 회사 설립·채용 (1단계~)

- [ ] CEO 회사 설립 (이름·아이콘·상품 등록)
- [ ] CEO → 직원 직접 채용 (최대 4명, 정원 초과 차단)
- [ ] 교사 → 공무원 임명
- [ ] 채용 마감 경고·긴급 배정
- [ ] 사업계획서 수정 1회 (2단계 진입 후)

### 6순위: 생산·급여·시설·공무원 시찰 (2단계~)

- [ ] DB 마이그레이션: products, facilities, facility_uses, inspection_reports, officer_alerts
- [ ] 직원 업무일지 (실물 제작 기록)
- [ ] CEO → 직원 급여 확인·지급 (멱등)
- [ ] 공무원 시설 목록 관리·이용료 설정
- [ ] 시설 이용 계산서 (CEO → 시청 이체)
- [ ] 공무원 시찰 보고서 (예산 경보·정체 경보 트리거)
- [ ] 교사 화면: 시찰 보고서 열람

### 7순위: 판매·QR 결제 (4단계)

- [ ] DB 마이그레이션: transactions (결제 타입)

## 주요 결정 사항 (재논의 불필요)

| 결정 | 내용 |
|---|---|
| 호스팅 | Vercel (무료) |
| DB | Supabase (무료, keep-alive cron으로 7일 정지 우회) |
| QR 방식 | 체크카드 방식 (손님 제시 → 판매자 스캔) |
| 박람회 | 교류의 장 (돈 거래 없음) |
| 임금 | CEO 4,500 / 공무원 3,500 / 직원 3,000 |
| CEO 역할 전직 | 다음 로그인 시 (reveal_pending 플래그) |
| 직원·공무원 전직 | 채용·임명 즉시 Realtime 푸시 |
| 공무원 채용 주체 | 교사(시장)가 임명 |
| 직원 채용 주체 | CEO가 직접 채용 |
| 이식성 | GitHub 포크 → Supabase 마이그레이션 → Vercel 배포 (약 30분) |
