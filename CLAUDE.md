# AI 작업 컨텍스트 — 경기 상생 창업 프로젝트

> Claude Code 및 다른 AI(GPT-4, Gemini 등)가 컨텍스트 없이 이 프로젝트를 이어서 작업할 때 읽는 파일.
> 작업 시작 전 **이 파일 → `docs/base_idea/웹앱_요구사항_사양서_v4.md` → `docs/PROGRESS.md`** 순서로 읽을 것.
> 불명확한 점이 있으면 코드 작성 전에 먼저 질문할 것.

---

## 프로젝트 한 줄 요약

초등 4학년 사회 수업용 창업 시뮬레이션 웹앱. 교사(시장)가 수업 단계를 통제하고, 학생이 CEO·직원·공무원 역할로 창업→생산→교류→판매를 경험한다. 모든 활동이 과정중심평가 데이터로 자동 기록된다.

---

## 절대 바꾸지 말 것

1. **기능 = 개념 정착 장치 세트**: 기능 구현 시 해당 개념 형성평가·micro-concept를 함께 심는다
2. **단계 잠금**: 학생은 교사가 연 단계 기능만 접근. 서버·클라이언트 이중 검증
3. **앱이 단일 진실 공급원**: 모형돈 병행 없음. 모든 거래는 앱 DB에만 기록
4. **방어 장치는 기능과 함께**: 나중에 몰아서 추가 금지
5. **타이핑 최소화**: 초4 대상. 큰 버튼, 드롭다운, 번호 선택 방식

## 기술 스택 (확정, 변경 불가)

```
Next.js 14 (App Router) + TypeScript + Tailwind CSS

Vercel (호스팅)
GitHub (저장소)
qrcode + html5-qrcode (QR 생성·스캔)
```

---

## 역할 체계

| 역할 | DB값 | company_id | 설명 |
|---|---|---|---|
| 시장 | `mayor` | null | 교사. 단계 통제·전체 조회 |
| 지원자 | `applicant` | null | 초기 역할. 선정/채용 후 전직 |
| CEO | `ceo` | 있음 | 선정된 창업가. 직원 채용 주체 |
| 직원 | `staff` | 있음 | CEO가 채용. 생산 담당 |
| 공무원 | `officer` | **null** | 교사가 임명. 반(class) 단위 소속 |

역할 전직 타이밍:
- `applicant → ceo`: 교사 선정 → `users.reveal_pending = 'ceo'` → **다음 로그인 시** 팝업
- `applicant → staff`: CEO 채용 → Supabase Realtime → **즉시** 팝업
- `applicant → officer`: 교사 임명 → Supabase Realtime → **즉시** 팝업

---

## 수업 단계

```
⓪ 도시탐구(0) → ① 창업(1) → ② 생산(2) → ③ 교류(3) → ④ 판매(4)
```

- `classes.stage` (0~4). 교사만 변경 가능
- 단계 변경 시 Supabase Realtime으로 전체 학생 화면 즉시 갱신
- 성찰·개념 작성은 모든 단계에서 항상 열림

## 임금 체계 (확정)

CEO 4,500원 / 공무원 3,500원 / 직원 3,000원 — 수업일당 1회, 멱등 처리 필수

---

## QR 결제 (확정: 체크카드 방식)

손님 QR 카드 제시 → 판매자 태블릿 스캔 → 상품 선택 + "왜 샀나요?" → 손님 PIN 입력 → 완료  
자가구매 차단(서버), 재고 초과 차단, 이중처리 방지(멱등키) 필수

---

## 박람회 = 교류의 장 (판매 아님)

돈 거래 없음. UI에서 "판매/구매/결제" 단어 절대 사용 금지. "제공/요청/협력 기록"만 사용

---

## 공무원 핵심 (v4 확장)

앱 알림 → 물리 이동 → 기업 시찰 보고서 작성 → 시장 제출  
신규 테이블: `inspection_reports`, `officer_alerts`

## 방어 장치 체크리스트

- [ ] 금액·수량: 양의 정수만 (서버 검증)
- [ ] PIN 3회 실패 → 30초 쿨다운
- [ ] 지원금 지급: 회사당 1회 (멱등)
- [ ] 급여: 수업일당 1회 (멱등)
- [ ] 결제: 거래 UUID 중복 거부
- [ ] 자가구매: 서버에서 차단
- [ ] 자유 입력: 금칙어 필터
- [ ] 개인정보: 번호·닉네임만. 실명·생년월일 수집 금지
